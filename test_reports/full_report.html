🐛 Bugs Found & Fixed During Audit

BUG 1 — Agency plan invisible on pricing page
src/app/page.tsx pricing grid was hardcoded to ['free', 'basic', 'pro']. Agency ($79/mo) never rendered.
Fix: Changed to ['free', 'basic', 'pro', 'agency'] with sm:grid-cols-2 lg:grid-cols-4.
BUG 2 — Stripe crash at build time
src/lib/stripe.ts called new Stripe(process.env.STRIPE_SECRET_KEY!, ...) at module load. Build fails with missing key.
Fix: Lazy initialisation via Proxy pattern — key only read at first call, throws with clear message.
BUG 3 — Webhook silently swallowed Supabase errors
src/app/api/stripe/webhook/route.ts fell back to anon key when service role key missing, silently processing events with wrong permissions.
Fix: getAdminClient() throws immediately if SUPABASE_SERVICE_ROLE_KEY not set.
BUG 4 — customer.subscription.updated not handled
Webhook only handled checkout.session.completed and deletion. Plan changes via Stripe Customer Portal never updated the DB.
Fix: Added handler that maps updated price ID → plan name via PLAN_PRICE_MAP.
BUG 5 — Default UI language was French, not English
src/contexts/UILanguageContext.tsx had useState('fr'). First-time visitors saw the site in French. E2E tests failed because English strings weren't found.
Fix: Changed to useState('en'); localStorage still persists user preference.
BUG 6 — DemoModal received non-existent isOpen prop
src/app/page.tsx passed isOpen={showDemo} but DemoModal.Props only has onClose. TypeScript build error.
Fix: Used conditional rendering: {showDemo && <DemoModal onClose={...} />}.
BUG 7 — "3 languages" copy across billing page
src/app/(dashboard)/billing/page.tsx had "3 languages / 3 langues / 3 idiomas / 3种语言 / 3 لغات" but the product supports 5 languages.
Fix: Global replace to "5 languages" in all 5 language variants including Arabic Unicode.

⚠️ Known Risks — Documented, Not Yet Fixed

RISK 1 — extra_credits not wired into generate route
Content Pack purchases add 20 credits to profiles.extra_credits via webhook, but /api/generate only checks plan.generations. Users who buy content packs are still blocked at their plan limit.
LOW URGENCY No revenue impact yet; users can't buy packs without knowing the limit.
RISK 2 — Approval tokens have no TTL
Email approval links use UUID v4 tokens (122-bit entropy). Brute force is infeasible, but intercepted tokens never expire.
LOW RISK Acceptable for MVP; consider 72h TTL in v2.
RISK 3 — Account reuse on same email
A user who deletes their account and re-signs-up with the same email gets a fresh generation count. Supabase Auth creates a new user_id.
LOW RISK Not easily exploitable; monitor via billing anomaly script.
RISK 4 — Content pack double-credit on webhook replay
checkout.session.completed for content_pack is not idempotent against duplicate webhook deliveries. A replayed event would add 20 credits twice.
LOW RISK Stripe rarely replays; implement session-ID dedup when content packs scale.
